AR Changelog

版本日誌

這裡詳細記錄每次修改訊息對應的子版本。除非指定更換版本號，後續修改會繼續以 v0.1a1-X 往下增加。

v0.1a1

2026 06/15
  • 新增 ar.html，作為 GitHub Pages 可直接開啟的 WebAR 頁面。
  • 新增 ar/ar-scene.js，使用 MindAR + Three.js 建立 image target 錨點場景。
  • 加入花瓣雨、花朵生長、柔和地面光斑與掃描提示文字。
  • 預留 PETAL_COUNT、FLOWER_COUNT、PETAL_FALL_SPEED、FLOWER_GROW_SPEED 等調整參數。
  • 新增 ar/assets/secret-garden-venue.jpg，由演出場地 HEIC 照片壓縮轉檔而成。

v0.1a1-1

2026 06/15
  • 調整 ar.html 為 import map 載入方式，讓 MindAR 能解析 three module。
  • 補上 three/addons/ 對應路徑，修正 MindAR 內部載入 CSS3DRenderer 時的 module specifier 問題。

v0.1a1-2

2026 06/15
  • 新增 AR 啟動前檢查：HTTPS、安全相機 API、targets.mind 大小與格式初步驗證。
  • 修正啟動失敗時只顯示籠統訊息的問題，改為顯示更精準的中文提示。

v0.1a1-3

2026 06/15
  • 新增原生相機預先授權流程，先確認瀏覽器能開啟後鏡頭，再交給 MindAR 啟動。
  • 修正部分瀏覽器回傳空錯誤時，狀態文字只出現「原始錯誤：」的問題。

v0.1a1-4

2026 06/15
  • 依手機截圖判斷頁面尚未觸發 onTargetFound，新增辨識逾時提醒，協助確認是否真的掃描成功。
  • 把花瓣雨與花朵座標縮回 image target 附近，避免辨識成功後物件落在手機視野外。
  • 新增中心柔光標記，辨識成功時會先看到一個淡色光圈，方便確認 AR 物件已被渲染。
  • 重新整理版本紀錄規則：主版本維持 v0.1a1，細部修改改記為 v0.1a1-X。

v0.1a1-5

2026 06/15
  • 新增固定顯示的 AR Debug Log 面板，記錄頁面載入、target 檔讀取、相機授權、MindAR 啟動、目標找到與目標遺失事件。
  • 新增 debug-counter，每次觸發 onTargetFound 會累加次數，方便現場確認是否真的掃描成功。
  • 在 image target anchor 上新增半透明紅色方塊與白色邊框，辨識成功後應立即出現在目標圖片位置。
  • 更新 ar/ar-scene.js 快取版本至 20260615-ar5。

v0.1a1-6

2026 06/15
  • 依排查建議，主 AR 頁暫時移除 requestCameraAccess() 預先相機授權呼叫，改由 MindAR 自行處理相機。
  • 主 AR 頁暫時移除 filterMinCF、filterBeta、uiLoading、uiScanning、uiError 等自訂參數，回到最乾淨的 MindARThree 設定。
  • 新增根目錄 ar-test.html，用最小 Three.js 紅色 cube 測試 ar/targets.mind 是否能被 MindAR 辨識。
  • 保留必要的 three/addons/ import map，避免 MindAR 內部載入 CSS3DRenderer 時產生 module 解析錯誤。
  • 更新 ar/ar-scene.js 快取版本至 20260615-ar6。

v0.1a1-7

2026 06/16
  • 確認 MindAR 1.2.5 的 MindARThree 沒有直接暴露 videoSettings，內部預設只要求 facingMode: "environment"。
  • 新增高畫質相機 wrapper，於 MindAR 啟動時把 getUserMedia video constraints 合併為 1920x1080、30fps 的 ideal 請求。
  • 新增實際相機設定 log：主 AR 頁會在 debug log 與 console 顯示實際取得的 width、height、frameRate、facingMode。
  • ar-test.html 也同步加入相機解析度 log，並在畫面底部顯示目前取得的相機解析度。
  • 更新主 AR 快取版本至 20260616-ar7。

v0.1a1-8

2026 06/16
  • 依已驗證可用的 ar-basic.html，將主 AR 頁改為 module 載入後即建立 MindARThree、addAnchor(0) 與 AR 物件。
  • 主 AR 頁按下開始後改為直接呼叫 await mindarThree.start()，並使用 renderer.setAnimationLoop() 渲染，對齊 ar-basic.html 的啟動流程。
  • 暫時移除主 AR 啟動流程中的 target fetch 檢查、高畫質相機 wrapper、重新建立 renderer/container 等差異點。
  • 將主頁 import map 的 Three.js 來源改為與 ar-basic.html 相同的 unpkg.com。
  • 更新主 AR 快取版本至 20260616-ar8。

v0.1a1-9

2026 06/16
  • 新增 ar/Red.mind，主 AR 頁改以 Red target 作為目前掃描目標。
  • 新增 ar/assets/red_bloom.mp3，掃描 Red target 時播放原秘密花園網站紅色花朵綻放音效。
  • 新增紅色主花 AR 綻放動畫：莖由下往上生長、八片紅色花瓣依序打開、花心彈出，對應原網站「花朵綻放」的動作節奏。
  • Red target found 時會重置並重新播放紅色開花動畫，同步寫入 debug log。
  • 更新主 AR 快取版本至 20260616-ar9。
